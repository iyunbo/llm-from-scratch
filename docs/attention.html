Attention Mechanisms

Chapter 3 — Three implementations of attention, built incrementally from raw matrix operations to the full multi-head version used in GPT.

The Core Formula

Attention(Q, K, V) = softmax(QKT / √dk) · V

Each token produces a Query (what am I looking for?), Key (what do I contain?), and Value (what do I transmit?). The dot product of Q and K measures relevance; softmax normalizes it into weights; the weighted sum of V produces the output.

Three Progressive Implementations

1. SimpleAttention

No learnable parameters. Manually computes Q, K, V projections using raw matrix multiplication to demonstrate the core attention math.

2. CausalAttention

Adds nn.Linear projections and a causal mask — an upper-triangular matrix that prevents each token from attending to future positions. This is what makes autoregressive generation possible.

# Causal mask: upper triangle filled with -inf
mask = torch.triu(torch.ones(seq_len, seq_len), diagonal=1)
scores = scores.masked_fill(mask.bool(), float('-inf'))

3. MultiHeadAttention

Runs multiple attention heads in parallel. Each head operates on a slice of the embedding dimension, learning different attention patterns. Outputs are concatenated and projected back.

Why Separate Q, K, V?

Decoupling the three projections lets the model learn what to match on (Q, K) independently from what information to transmit (V). A token might match on syntactic role but transmit semantic content.

Why Scale by √dk?

Without scaling, large dot products push softmax into saturation (near-zero gradients). Dividing by √dk keeps the variance of scores roughly constant regardless of dimension.

Key Design Choices

Causal Mask Prevents attending to future tokens — essential for autoregressive generation
Attention Dropout Prevents over-reliance on fixed positions; improves generalization
Head Specialization Different heads learn different patterns: syntax, semantics, position
Same Parameter Count Multi-head uses the same total parameters as single-head but is more expressive
