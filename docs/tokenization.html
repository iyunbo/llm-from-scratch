Text Data & Tokenization

Chapter 2 — Converting raw text into the numerical sequences that a neural network can process.

Pipeline Overview

Tokenization

The project implements two tokenizers: a simple vocabulary-based one for learning, and OpenAI's BPE tokenizer via tiktoken for production use. BPE (Byte Pair Encoding) iteratively merges the most frequent character pairs, building a vocabulary of 50,257 subword tokens that can represent any text.

import tiktoken
enc = tiktoken.get_encoding("gpt2")
ids = enc.encode("Hello, world!")  # [15496, 11, 995, 0]

Embeddings

Each token ID maps to a dense vector via nn.Embedding. A separate positional embedding encodes where each token sits in the sequence. The two are summed to produce the input to the transformer.

input = token_embedding(x) + position_embedding(positions)
ComponentShapePurpose
Token embedding50,257 × 768Map token IDs to dense vectors
Position embedding1,024 × 768Encode sequence position

Sliding Window DataLoader

Training data is created by sliding a window across the tokenized text. Each window produces an (input, target) pair where the target is the input shifted by one position — the classic next-token prediction setup.

# stride < max_len creates overlapping windows
# "The cat sat on" -> input: [The, cat, sat], target: [cat, sat, on]
dataset = GPTDataset(tokens, max_len=4, stride=2)
