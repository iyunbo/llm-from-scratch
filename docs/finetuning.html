Fine-tuning

Chapters 6 & 7 — Adapting the pretrained model for classification and instruction-following.

Classification (Chapter 6)

Transfer learning: load the pretrained GPT backbone, freeze most weights, and add a classification head that reads the last token's hidden state.

Why the Last Token?

In a causal (left-to-right) model, the last token has attended to every previous token through the self-attention layers. It's the most "informed" position in the sequence — making it the natural choice for classification.

Freeze Strategies

StrategyTrainable %Use Case
Head only~0.03%Small dataset, strong pretrained weights
Last N blocks + head~10%Medium dataset, domain shift
Full fine-tuning100%Large dataset, significant domain shift

Instruction Tuning (Chapter 7)

Train the model to follow instructions using the Alpaca format — structured prompts with instruction/response pairs.

Prompt Format

### Instruction:
Explain the difference between a list and a tuple in Python.

### Response:
A list is mutable and uses square brackets []. A tuple is
immutable and uses parentheses (). Lists are for collections
that change; tuples are for fixed data.

Training Details

The instruction dataset wraps each example in this format. Training uses cross-entropy loss on the full sequence (a simplified approach; production systems mask the loss on the instruction portion so the model only learns to generate responses).

Interactive Chat

Chapter 7 includes a REPL interface for chatting with the trained model:

python -m ch07.chat

# Commands inside the chat:
#   temperature 0.8   - adjust randomness
#   top_k 40          - adjust sampling pool
#   quit              - exit

User input is wrapped in the instruction prompt template, and the model generates a response using top-k sampling with temperature.
