GPT Model Architecture

Chapter 4 — Assembling transformer blocks, activation functions, and normalization into a complete GPT model.

Model Stack

The GPT model is a stack of identical transformer blocks, bookended by embeddings at the input and a linear projection at the output.

class GPTModel:
    token_emb + pos_emb      # Input embeddings
    dropout
    TransformerBlock × N     # Repeated N times
    LayerNorm                 # Final normalization
    Linear(d_model, vocab)    # Output head

Transformer Block

Each block uses pre-norm architecture with residual connections:

x = x + Attention(LayerNorm(x))
x = x + FeedForward(LayerNorm(x))

Pre-norm (normalizing before the sublayer) is more numerically stable than post-norm for deep networks, which is why modern LLMs use it.

Components

LayerNorm

Normalizes activations across the feature dimension with learnable scale (gamma) and shift (beta) parameters. Keeps gradients well-behaved through deep networks.

GELU Activation

Gaussian Error Linear Unit — a smooth, probabilistic gate that lets some negative values through (unlike ReLU's hard cutoff). Standard in modern transformers.

GELU(x) = x · Φ(x)    where Φ is the Gaussian CDF

FeedForward Network

A position-wise MLP that expands to 4× the model dimension, applies GELU, then projects back:

Linear(d_model, 4 * d_model)  # Expand
GELU()                         # Activate
Linear(4 * d_model, d_model)  # Compress
Dropout()

Text Generation

Two strategies for sampling the next token from the output distribution:

MethodHow It WorksTrade-off
GreedyAlways pick the highest-probability tokenDeterministic but repetitive
Top-k + TemperatureSample from the k most likely tokens, with temperature scalingMore diverse and natural

Temperature < 1 sharpens the distribution (more confident picks); temperature > 1 flattens it (more creative/random).
